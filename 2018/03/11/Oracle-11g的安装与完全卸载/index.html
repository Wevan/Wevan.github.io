Oracle 11g的安装与完全卸载

发表于 2018-03-11 | | 阅读次数:

安装

准备工作

首先先去官网下载安装包，下载地址,下载之前需要注册一个Oracle账户。完成后在下载页面找到11g Release，win64版本下载，下载的文件一共有两个，File1和File2

下载完成后，将他们放在一个空文件夹中，然后选中两者同时解压到一个新的文件夹下。

开始安装

在解压后的文件夹中找到setup.exe启动即可

启动setup.exe时可能会碰到弹框环境不符合条件的情况，修改database\stage\cvu\cvu_prereq.xml：

保存后重新启动setup.exe即可

取消勾选该项，下一步，是

此选项安装完数据库管理软件后，系统会自动创建一个数据库实例

默认桌面类，因为我们只是个人电脑安装，电脑不做专业版服务器使用

Oracel建议的密码规则必须是大写字母加小写字母加数字，不能含有符号， 8位以上。不符合Oracel建议的密码也可以忽略其警告。其余默认配置就好。

直接是就好

也许会遇到检查环境不合标准的情况，可直接忽略

但是也可以解决，如Path这个不符合需求，在早期的windows版本中，会对这个做限制，但是Oracle的安装也会。解决办法是，先把Path中的配置拷贝出来，删一部分后再放进去，重启电脑，重新打开安装程序，安装完成后再将Path的值全部复制回去复原。重启即可。

数据库管理软件文件及dbms文件安装完后，会自动创建安装一个实例数据库默认前面的orcl名称的数据库。

除了系统默认的system账户，我们还可以解锁其他账户如sctto账户，点击口令管理设置即可，需要设置输入新账户的密码

IDEA连接Oracle

修改SID为orcl

测试一下

卸载

任务管理器中关闭所有Oracle服务

在开始菜单中，找到Universal Installer，运行Oracle Universal Installer，单击卸载产品
在产品清单窗口中，单击全部展开，除了OraDb11g_home1外，勾选其他项目，单击删除

打开Cmd，执行regedit，打开注册表编辑器，删除 HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\ 下所有Oracle开头的服务。这个键是标识Oracle在windows下注册的各种服务。

删除计算机\HKEY_LOCAL_MACHINE\SOFTWARE\Oracle

删除注册的oracle事件日志：删除计算机\HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\EventLog\Application下所有以Oracle开头的项目（图中已删除一部分）

删除环境变量path中关于oracle的内容

重新启动操作系统。

重启后可以在任务管理器看到Oracle相关的进程不会再加载了（可将进程按名称正序排列后方便查看）

删除oracle安装目录下的所有数据

完成后删除C:\Program Files下oracle目录。

.删除开始菜单下oracle项，如：
C:\User（用户）\用户名\Appdata\Roaming\Microsoft\Windows\Start Menu（开始菜单）\Programs（程序）

有时在用户名目录下有Oracle的文件了，删除即可

不删除开始菜单下的Oracle相关菜单目录也没关系，不会影响再次安装Oracle。当再次安装Oracle时，该菜单会被替换。
